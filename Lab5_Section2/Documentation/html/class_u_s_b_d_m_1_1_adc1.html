Configure the ADC.

Parameters
[in]adcResolutionResolution for converter e.g. AdcResolution_16bit_se
[in]adcClockSourceClock source e.g. AdcClockSource_Asynch
[in]adcSampleInput sample interval. Allows use of higher input impedance sources
[in]adcPowerAllows reduced power consumption but with restricted input clock speed
[in]adcMuxselSelects between A/B multiplexor inputs on channels 4-8
[in]adcClockRangeAllows higher input clock speed operation
[in]adcAsyncClockControls whether the internal ADC clock is always enabled or only when needed for a conversion
Note
These settings apply to all channels on the ADC.
The resulting ADC clock rate should be restricted to the following ranges (assumes AdcPower_Normal, AdcClockRange_high):
[2..12MHz] for 16-bit conversion modes
[1..18MHz] for other conversion modes

◆ enableAsynchronousClock()

static void USBDM::AdcBase_T< Adc1Info >::enableAsynchronousClock ( )
inlinestaticinherited

Enable ADC internal asynchronous clock source.

Note
It is not necessary to enable the internal clock to use it as an ADC clock source.
If the internal clock is selected, it will be automatically enabled when an ADC conversion is initiated.
However, enabling it beforehand will reduce the latency of the 1st conversion in a sequence.

◆ setCallback()

static void USBDM::AdcBase_T< Adc1Info >::setCallback ( ADCCallbackFunction  callback)
inlinestaticinherited

Set callback for conversion complete interrupts.

Parameters
[in]callbackCallback function to execute on interrupt.
Use nullptr to remove callback.
Returns
E_NO_ERROR No error
E_HANDLER_ALREADY_SET Handler already set
Note
There is a single callback function for all channels of the ADC.
It is necessary to identify the originating channel in the callback.
To change between handlers first use setCallback(nullptr).
